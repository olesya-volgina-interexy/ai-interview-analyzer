# Руководство пользователя — AI Interview Analyzer

Приложение для автоматического анализа интервью с кандидатами через LLM. Позволяет отслеживать воронку найма, собирать аналитику по клиентам/ролям/менеджерам и автоматически получать анализы по тикетам из Linear.

## Общая навигация

**Верхняя панель (header):**
- Логотип **Interview AI** слева;
- Три вкладки: **Dashboard**, **Interviews**, **Candidates**;
- Кнопка **+ New Analysis** — открывает форму анализа вручную;
- **Селектор периода** (иконка календаря) — фильтрует метрики на Dashboard. Варианты: текущий месяц (по умолчанию), прошлый месяц, последние 3 месяца, год, произвольный диапазон.

> Селектор периода влияет **только на Dashboard**. На страницах Interviews и Candidates у каждой свои фильтры.

## Страница Dashboard

Главная страница с общей аналитикой за выбранный период.

### Верхний ряд KPI-карточек

| Карточка | Что показывает |
|---|---|
| **Total Interviews** | Общее число интервью за всё время + разбивка по стадиям (manager call / technical) |
| **Hire Rate** | Процент найма среди технических интервью |
| **Avg Score** | Средний AI-скор (0-100) среди технических интервью |
| **Top Role** | Наиболее часто анализируемая роль |

> KPI-блок всегда показывает all-time статистику и не реагирует на селектор периода — это «пульс» системы в целом.

### Вкладка **Overview**

Две диаграммы-распределения по всем интервью: bar-chart по ролям и pie-chart по стадиям.

### Вкладка **Pipeline**

Воронка найма за период:

- **Requests Stats** — количество входящих тикетов по клиентам и по статусам (Triage, In Progress, Client Review, Manager Call, Technical, Hired, Rejected, On Hold). Есть кнопка **Refresh** — принудительно пересчитать (сбросить кэш).
- **Pipeline Funnel** — столбчатая воронка CV Sent → Manager Call → Technical → Final Result → Hired. Внизу: проценты конверсии `Manager Call → Technical` и `Technical → Hired`, а также общее число отправленных CV.
- **Time on Stages** — верхний акцентный блок показывает среднее время от появления тикета до найма (`Avg time from Triage to Hired`). Ниже — сегментированная полоса прогресса по стадиям (Triage / In Progress / Client Review / Broker's Call / Tech Call) с подписями времени в днях. Под ней — тренд количества интервью по месяцам.

### Вкладка **Quality & Insights**

- **Quality Stats** — топ-кластеры причин отказа (`Decision Breakers`), слабостей кандидатов (`Weaknesses`), внешнего фидбека от клиентов (`External Reasons`) и hire rate по ролям. Текстовые причины кластеризуются через LLM.
- **Role Scores** — средний скор по вакансиям.
- **Level Scores** — средний скор по уровням (Junior / Middle / Senior / Architect).

### Блок **Recent Analyses**

5 последних интервью с кликабельными строками — открывают модалку с полным анализом.

## Страница Interviews

Полная история всех проведённых анализов интервью с фильтрами.

### Фильтры (строка сверху)

| Фильтр | Поведение |
|---|---|
| **Role** | Точное совпадение с ролью |
| **Level** | Junior / Middle / Senior / Architect |
| **Stage** | Manager Call / Technical / Final Result |
| **Decision** | Hired / Rejected |
| **Manager** | Выбор из реально встречавшихся имён менеджеров |
| **Search by client...** | Подстроковый поиск по названию клиента (регистронезависимо) |
| **Clear** | Сбросить все фильтры |

### Таблица

Колонки: Date, Stage, Role, Client, Manager, Result (badge), Score. Сортировка по дате (свежие сверху). Клик по строке открывает модалку интервью.

## Страница Candidates

Список всех кандидатов с агрегированной статистикой.

### Поиск и фильтры

- **Search by name** — текстовый поиск по имени, c debounce 300 мс;
- **All Roles** — фильтр по роли, на которую кандидат проходил интервью;
- **All Results** — Hired (есть хоть один найм) / Not Hired (успешных найма нет);
- **Clear** — сбросить фильтры.

### Таблица

| Колонка | Смысл |
|---|---|
| **Candidate** | Имя |
| **Roles** | Роли, на которые кандидат проходил интервью (до 2 видно, остальные под «+N») |
| **Interviews** | Всего интервью (сортируемая колонка) |
| **Hired** | Сколько раз кандидат был нанят (в разных тикетах) |
| **Rejected** | Сколько раз получен отказ |
| **Score** | Средний скор (сортируемая колонка) |
| **Last** | Дата последнего интервью (сортируемая колонка) |

**Пагинация:** 20 записей на страницу, кнопки Previous / Next.

> `Hired` / `Rejected` у кандидата считаются только по **финальной стадии** тикета (когда тикет в Linear переходит в `Hired` / `Lost`). Стадии Manager Call и Technical в этот счёт не идут до закрытия тикета.

## Страница Candidate Detail

Открывается по клику на кандидата в таблице или напрямую по URL `/candidates/:name`.

### Заголовок

Кнопка возврата, имя кандидата, список ролей и суммарное число интервью.

### Stat-карточки (5 колонок)

- **CV Submitted** — сколько раз отправляли CV этого кандидата (сумма `cvSentCount` по всем связанным тикетам);
- **Total Interviews** — всего интервью;
- **Hired** / **Rejected** — см. выше;
- **Avg Score** — среднее по техническим скорам.

### Insight-карточки (3 колонки)

- **Top Strengths** — сильные стороны, кластеризованные LLM по всем интервью кандидата (тэги с частотой);
- **Top Weaknesses** — слабости;
- **Decision Breakers** — критические причины отказа.

### История интервью

Таблица со столбцами: Date, Stage, Role, Client, Manager, Result, Score. Клик открывает модалку конкретного интервью.

## Создание анализа вручную («+ New Analysis»)

Кнопка в верхней панели открывает модалку с формой. Нужна для разовых анализов — когда в Linear анализ не был запущен автоматически.

### Поля формы

**Transcript** *(обязательно)*
Текстовое поле под транскрипт интервью (минимум 200 px высоты). Поддерживает drag-and-drop файла `.txt`.

**Interview Details** *(обязательно)*
- **Stage** — Manager Call / Technical;
- **Role** — Backend / Frontend / Fullstack / DevOps / QA / Mobile;
- **Level** — Junior / Middle / Senior / Architect;
- **Interviewer Decision** — появляется только при `Stage = Technical`: Hired / Rejected.

**Participants** *(опционально)*
- **Client Name** — название клиента;
- **Candidate Name** — имя кандидата (если не указать — возьмётся из CV/транскрипта);
- **Manager** — имя проводившего интервью.

**Дополнительно** *(опционально)*
- **Broker Request** — требования от брокера;
- **Interviewer Comments** — комментарии менеджера;
- **Linear Issue ID** — например, `ENG-123`, чтобы связать с тикетом в Linear.

### Запуск и прогресс

Кнопка **Start Analysis** запускает анализ. Появляется прогресс-экран с этапами:

1. Downloading CV;
2. Creating embedding;
3. Finding similar cases;
4. AI analysis;
5. Saving results;
6. Done.

По завершении — три кнопки:
- **New Analysis** — обнулить форму и начать заново;
- **Close** — закрыть модалку;
- **View in History →** — открыть страницу Interviews.

## Модалка интервью (Analysis, Transcript, CV, Questions)

Открывается кликом по строке в любой таблице (Dashboard → Recent Analyses, Interviews, Candidate Detail).

### Заголовок

Аватар с именем кандидата, роль, уровень, клиент, дата. Кнопки:
- **Save as PDF** — экспорт отчёта (печать);
- **Delete** — удалить запись (с подтверждением);
- **×** — закрыть.

### Вкладки (зависят от наличия данных)

**Analysis** — AI-оценка, форма зависит от стадии:

- **Manager Call:**
    - Бейдж рекомендации (Passed / Rejected / On Hold);
    - **Overview** — общая оценка + сильные/слабые стороны;
    - **Soft Skills** — коммуникация, мотивация, culture fit, зарплатные ожидания, ясность мышления;
    - **Broker Fit** — соответствие требованиям брокера;
    - **Decision** — обоснование рекомендации.

- **Technical:**
    - Бейдж рекомендации (Hire / No Hire / Uncertain);
    - Индикатор уровня + полоска Score (зелёная ≥75, янтарная ≥50, красная <50);
    - **Overview**, **Skills Match**, **Broker Match**, **Decision**.

- **Final Result:**
    - Индикатор Hired / Lost;
    - Сводка по manager call и technical;
    - Итоговая рекомендация.

**Transcript** — сырой транскрипт в моноширинном шрифте.

**CV** — текст извлечённого CV.

**Broker Request** — исходный текст требований клиента.

**Questions** — список вопросов, заданных в ходе интервью, с тэгом «как кандидат ответил»: Well / Partial / Poor.

## Интеграция с Linear

Большая часть интервью анализируется автоматически через webhook Linear. Пользователь работает с тикетами в Linear, а приложение реагирует на хэштеги в комментариях и смены статуса.

### Триггеры в комментариях тикета

| Хэштег | Когда срабатывает | Что происходит |
|---|---|---|
| **#manager_call_transcript** или **#feedback_manager_call** | Статус тикета = `Broker's Call` | Извлекается нужный кандидат из треда, подтягиваются CV и транскрипт, запускается анализ стадии `manager_call`. Если у этого кандидата уже есть анализ manager_call — запуск пропускается. |
| **#technical_call_transcript** | Статус тикета = `Tech Call` | Подтягивается транскрипт технички, запускается анализ стадии `technical`. Повторные запуски не дублируются. |
| **#feedback** *(без суффикса `_manager_call`)* | Любой статус | Текст комментария записывается как `externalFeedback` тикета (причина отказа со стороны клиента). Попадает в `Top External Reasons` на дашборде. |
| Ссылка `my.visualcv.com` в комментарии | Любой статус | Счётчик `cvSentCount` у тикета увеличивается, статус меняется на `cv_sent` (если ещё не дальше по воронке). |

### Триггеры смены статуса тикета

| Новый статус | Действие |
|---|---|
| **Triage** → **In Progress** → **Client Review** → **Broker's Call** → **Tech Call** | Внутренний статус тикета синхронизируется, добавляется строка в `IncomingRequestStatusHistory` для расчёта Time On Stage. |
| **Hired** | Создаётся анализ `final_result` с `decision = 'hired'` **только для тех кандидатов, в ветке которых есть reply `#hired`**. Тикет считается успешно закрытым. |
| **Lost** | То же, но `decision = 'rejected'` и только для кандидатов с reply `#lost` в их ветке. |
| **On Hold** | Только смена статуса, без создания анализа. |

### Смена заголовка тикета

Если в заголовке тикета меняется имя клиента (форматы `Role for Company` или `Role - Company`), приложение обновит `clientName` у соответствующего `IncomingRequest` и у всех уже проанализированных интервью в этом тикете. Если в новом заголовке клиента нет — обновление пропускается, старое значение сохраняется.

### Структура треда кандидата в Linear

Каждая ветка комментариев в тикете представляет одного кандидата. Ветка считается «веткой кандидата», если **в root-комментарии есть CV-ссылка** (visualcv) **или в ветке есть хотя бы один хэштег** (`#manager_call_transcript`, `#technical_call_transcript`, `#feedback_manager_call`, `#hired`, `#lost`).

## Частые задачи и подсказки

### «Хочу посмотреть, как долго идёт тикет до найма»
Dashboard → вкладка **Pipeline** → блок **Time on Stages**. Верхняя метрика — среднее время от Triage до Hired. Ниже — полоса с разбивкой по стадиям.

> Метрики считаются по тикетам, созданным **после** внедрения трекинга истории статусов. Старые тикеты в эти средние не попадают.

### «Нужно найти все интервью кандидата Ivan Petrov»
Вариант 1: вбить имя в поиск на странице **Candidates** → кликнуть по строке → открыть **Candidate Detail**.
Вариант 2: на странице **Interviews** использовать фильтры по роли/менеджеру/клиенту.

### «Не обновляется статистика на дашборде»
Нажать кнопку **Refresh** в блоке Requests Stats, либо открыть вкладку с параметром `?refresh=1` — сбросит Redis-кэш (TTL у него и так 30 минут).

### «На странице интервью поиск по клиенту не находит»
Поиск регистронезависимый и по подстроке — достаточно ввести часть названия (например, «acme» найдёт «ACME Corp»).

### «Тикет в Linear перешёл в Hired, но анализ не создался»
Проверить: (1) есть ли у этого кандидата в его ветке reply с текстом `#hired` (или `#lost` для `Lost`) — без него `final_result` не создаётся, даже если статус тикета сменился; (2) есть ли у ветки CV-ссылка в root-комментарии или хоть один хэштег — иначе ветка не распознаётся как кандидат. Если всё на месте и анализ всё равно не появился — обратиться к админу, посмотреть логи webhook.

### «Запустить анализ вручную, когда в Linear ничего нет»
**+ New Analysis** в шапке → заполнить транскрипт и детали → **Start Analysis**.

### «Посмотреть скоры по уровням — сравнить Junior и Senior»
Dashboard → вкладка **Quality & Insights** → карточка **Level Scores**.

### «Причины отказов — что чаще всего встречается»
Dashboard → вкладка **Quality & Insights** → **Decision Breakers** и **External Reasons** (внешний фидбек от клиента).
